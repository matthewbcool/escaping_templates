Servers

Servers are computers that provide data to other computers over some sort of network. We get to work with web servers which store website data for other computers to access. The protocol we use to communicate with servers on the web is HTTP Hyper Text Transfer Protocol. When someone wants to access a website, there device (the client) sends an HTTP request (ie: GET) to the server and the server retrives the data and responds. Responses can be static, such as a prewritten file or image or dynamic like our web app generating HTML. Here is a video about how servers work in web dev.

Validating Input

Validating input is about preventing the user from making mistakes, doing our best to create a secure information transaction and collecting data from user input the way we require it. If we don't validate input, it could have potential security implications for our servers or our websites. Users could break the website or potentially insert malicious code.

HTML Templates and Abstraction

HTML templates allow us to dynamically generate pieces of a webpage as we see fit. Templates help us to avoid repitition in the HTML we write and allow us to build more complex web applications more easily. For example, if your website has a header that you want to use across several webpages, you can use that header in a template rather then reusing the same code over and over across all your pages manually. Templates fall in line with the abstraction principal, as they help us to avoid duplication.
